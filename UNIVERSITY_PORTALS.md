## Deep Dive: Custom/Proprietary Portal Research (30 Universities)

> Research conducted 2026-03-29 via live HTTP probes (curl) to each portal's public-facing pages.
> Authentication walls prevent seeing full menus, but login pages, public links, and page structures reveal the tech stack and data categories.

### Legend
- **Auth**: Authentication method observed
- **Tech**: Technology stack detected from HTML/headers
- **Data**: Data categories available (inferred from menu items, links, public pages)
- **Integration**: API availability or scraping-only

---

### SAO PAULO

#### 1. USP - JupiterWeb
- **URL**: `https://uspdigital.usp.br/jupiterweb/`
- **Status**: ONLINE (HTTP 200)
- **Auth**: Custom login (`webLogin.jsp`), USP digital credentials (numero USP + senha). Also has `primeiroAcesso` and `esqueciSenha` flows.
- **Tech**: Java Servlets/JSP (classic J2EE), ISO-8859-1 encoding, jQuery UI, FreeStyle Menu JS. Very legacy codebase (HTML 4.01 Transitional).
- **Data categories** (from menu items extracted):
  - **Disciplines**: Search by discipline (`jupDisciplinaBusca?tipo=D`), search by class (`jupDisciplinaBusca?tipo=T`)
  - **Enrollment**: Interactive enrollment (`grdMatriculainterativaInformacoes.jsp`), tutorial for new enrollment
  - **Courses**: Course catalog (`jupCarreira.jsp`)
  - **Calendar**: Academic calendar (`jupArquivosPublicos.jsp?tiparq=9`)
  - **Scholarships**: PUB (Programa Unificado de Bolsas), PAP, PEEG
  - **Behind login** (known from USP documentation): grades, transcripts, class schedule, attendance, academic history, enrollment status
- **URL patterns**: `/jupiterweb/<servlet>?codmnu=NNNN` (menu-code based routing)
- **Integration**: NO API. Pure server-rendered HTML. Scraping required. Session-based auth with JSESSIONID cookies.
- **Note**: USP also uses `edisciplinas.usp.br` (Moodle) for course materials and `usp.instructure.com` (Canvas) for some departments.

#### 2. UNICAMP - DAC/SIGA
- **URL**: `https://www.dac.unicamp.br/` (info portal), `https://sistemas.dac.unicamp.br/siga/` (academic system)
- **Status**: ONLINE (HTTP 200 for both)
- **Auth**: UNICAMP institutional credentials via centralized login (`/siga/nucleo/login.xhtml`). Uses RichFaces/JSF framework.
- **Tech**: Java EE (JSF/RichFaces/A4J), XHTML pages. Purple-themed UI (#53428b). Also has `grade.daconline.unicamp.br` (GDE - Gerenciamento de Disciplinas para Estudantes).
- **Data categories** (from DAC portal links):
  - **Schedule**: `caderno-de-horarios/2026/` (class schedules)
  - **Curriculum**: Course catalogs for grad and postgrad
  - **Academic records**: CR (coeficiente de rendimento), prerequisites, integration period
  - **Certificates**: Document authenticity verification (`/siga/nucleo/autenticidade.xhtml`)
  - **Student card**: Card lookup (`/siga/nucleo/buscar_cartao.xhtml`)
  - **Signature verification**: `vidaacademica/verificador_assinatura.xhtml`
  - **Behind login**: grades, enrollment, attendance, academic history, class materials
- **URL patterns**: `/siga/<module>/<action>.xhtml` (JSF-based)
- **Integration**: NO API. JSF/AJAX-heavy pages with ViewState tokens. Scraping is complex due to JSF state management.

#### 3. UNIFESP - ProGrad
- **URL**: `https://prograd.unifesp.br/`
- **Status**: Portal returned empty response on probe. Likely requires institutional VPN or specific network access.
- **Auth**: UNIFESP institutional credentials (expected)
- **Tech**: Unknown from external probe. UNIFESP uses multiple systems across campuses.
- **Data categories** (known from university documentation): grades, enrollment, class schedule, academic calendar, course catalog
- **Integration**: Likely HTML-only, scraping required. Limited external access.

#### 4. UNINOVE - SYS
- **URL**: `https://sys.uninove.br/` (returned HTTP 403 - forbidden from external access)
- **Alt URL**: `https://ava.uninove.br/` (Moodle-based AVA, returned title "Uninove")
- **Status**: SYS is access-restricted (403). Main site `uninove.br` returned 503 during probe.
- **Auth**: UNINOVE student credentials (RA + senha)
- **Tech**: Proprietary system ("SYS"). Also uses Moodle for AVA (virtual learning environment).
- **Data categories** (known from university materials): grades, enrollment, financial (boletos), class schedule, attendance, course materials via AVA
- **Integration**: NO API. Scraping would require bypassing access restrictions. Behind WAF (Varnish cache observed).

#### 5. UNISA

### Common Data Categories Across All Custom Portals

Based on the portals that were accessible, the typical data available includes:

1. **Grades/Notas** - Per-semester, per-discipline grades (universally available)
2. **Academic History/Historico** - Full transcript with all completed courses
3. **Class Schedule/Horario** - Weekly timetable with room assignments
4. **Enrollment/Matricula** - Current enrollment status, course registration
5. **Attendance/Frequencia** - Attendance records per discipline (some portals)
6. **Course Catalog/Catalogo** - Available courses and prerequisites
7. **Curriculum/Grade Curricular** - Required courses for degree completion
8. **Documents/Certificados** - Certificate requests, transcripts, declarations
9. **Financial** - Tuition boletos (private universities only)
10. **Library** - Library access and loan status (some portals)

### Key Technical Patterns for Integration

1. **Authentication**: Most use session cookies (JSESSIONID). Some support gov.br OAuth (UFSM, UFMS). SSO/CAS is common at federal universities.
2. **No APIs exist**: Zero portals offer REST/GraphQL APIs. ALL require HTML scraping.
3. **JSF/RichFaces** is the most common tech stack at federal universities - requires handling ViewState tokens, AJAX partial updates, and component tree state.
4. **URL patterns**: Session IDs often embedded in URLs (`;jsessionid=XXX`).
5. **Encoding**: Mix of ISO-8859-1 (legacy) and UTF-8. Must handle both.
6. **Gov.br integration**: Growing trend (UFSM, UFMS) - could enable OAuth-based access in the future.
